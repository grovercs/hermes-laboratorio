# Hermes Laboratorio

Este repositorio sirve como laboratorio de pruebas y documentación para Hermes.

## Objetivo

Mantener una copia segura de contexto y documentación útil relacionada con el uso local de Hermes, sin subir credenciales ni información sensible.

## Contenido previsto

- `docs/`: documentación técnica, notas estructuradas y guías de uso.
- `backups/`: copias seguras de archivos de contexto permitidos, como `SOUL.md`.
- `notes/`: notas de laboratorio, pruebas y aprendizaje.

## Seguridad

Este repositorio NO debe contener:

- Contraseñas.
- Tokens.
- Claves API.
- Archivos `.env`.
- `auth.json`.
- `config.yaml` con credenciales.
- Sesiones de Hermes.
- Logs.
- Credenciales de clientes.
- Datos bancarios o información sensible.

Antes de subir cambios, revisar siempre `git status` y comprobar que no se está incluyendo ningún archivo sensible.
